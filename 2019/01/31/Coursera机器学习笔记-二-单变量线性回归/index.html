一、单变量线性回归是什么

单变量线性回归是一个数据的变化只与一个因素有关，并且是线性相关。回归一词指的是，我们根据之前的数据预测出一个准确的输出值。

回顾笔记(一)中预测卖房价格的例子。假如我们有一个1250平方尺大小的房子要出售，并且有一个包含卖房价格的数据集。那么要做的第一件事是构建一个模型。对于当前例子来说，模型就是房子的出售价格与房子尺寸大小的变化关系，它可能是一个线性关系。从这个数据模型上来看，也许你可以以大约220000(美元)左右的价格卖掉这个房子。

而在机器学习中，我们常称这个数据集为训练集。

我们将要用来描述这个回归问题的标记如下:

  • $m$ 代表训练集中实例的数量，即训练集数据总数
  • $x$ 代表特征/输入变量
  • $y$ 代表目标变量/输出变量
  • $\left( x,y \right)$ 代表训练集中的实例
  • $({x^{(i)}},{y^{(i)}})$ 代表第$i$ 个观察实例
  • $h$ 代表学习算法的解决方案或函数也称为假设

(二)假设函数

下图是一个监督学习算法的工作方式。程序在获得训练集后，进行一系列的学习算法，最后输出一个假设函数。我们通过假设函数$h$，根据房屋价大小$x$值，推测房屋能够卖出的价格$y$。因此，$h$ 是一个从$x$ 到 $y$ 的函数映射。那么，对于我们的房价预测问题，我们该如何表达 $h$？

一种可能的表达方式为：$h_\theta \left( x \right)=\theta_{0} + \theta_{1}x$，因为只含有一个特征/输入变量，因此这样的问题叫作单变量线性回归问题。

我们现在假定在上述问题中的假设函数是这样的线性函数形式:$h_\theta \left( x \right)=\theta_{0}+\theta_{1}x$。那么接下来需要做的就是为模型选择合适的参数$\theta_{0}$ 和 $\theta_{1}$。

二、代价函数

选择的参数决定了我们得到的直线相对于训练集的准确程度，模型所预测的值与训练集中实际之间的差距（下图中蓝线所指）就是建模误差。

我们的目标便是选择出可以使得建模误差的平方和能够最小的模型参数。 即使得代价函数 $J \left( \theta_0, \theta_1 \right) = \frac{1}{2m}\sum\limits_{i=1}^m \left( h_{\theta}(x^{(i)})-y^{(i)} \right)^{2}$最小，即所有数据的平均误差。其中$\frac{1}{2}$仅仅为了之后计算方便。

我们绘制一个等高线图，三个坐标分别为$\theta_{0}$和$\theta_{1}$ 和$J(\theta_{0}, \theta_{1})$：则可以看出在三维空间中存在一个使得$J(\theta_{0}, \theta_{1})$最小的点。

代价函数也被称作平方误差函数。我们之所以要求出误差的平方和，是因为误差平方代价函数，对于大多数问题，特别是回归问题，都是一个合理的选择。还有其他的代价函数也能很好地发挥作用，但是平方误差代价函数可能是解决回归问题最常用的手段了。

三、梯度下降

(一)梯度下降概念

上面我们提到了，为了使模型更符合实际，我们需要使得代价函数的值最小。而我们可以通过代价函数绘制出一个等高线图。而梯度下降算法就是求得等高线图中最低点的算法。

梯度下降，从名字我们就可以得知是从梯度入手。梯度下降的思想就是：开始时我们随机选择一个参数的组合$\left( {\theta_{0}},{\theta_{1}},……,{\theta_{n}} \right)$，计算代价函数，然后我们寻找下一个能让代价函数值下降最多的参数组合。我们持续这么做直到到达一个局部最小值。但由于初始参数组合的不同可能导致最后局部最小值不同，例如上图中的两条路径，因此我们有时需要尝试不同的初始参数组合。

为了更好的理解梯度下降算法，举个例子，想象我们站在山上某一点上。环顾四周，为了下山更快，我们选择最陡的方向下山。每走一步重新更新最陡的下山方向。就这样直到最后，我们发现四周都比自己所处的地方高，那么自己就处于一个局部最低点。梯度下降算法就是这个思想。

需要注意的是，在有多个参数的情况下，我们更新${\theta_{0}}$到${\theta_{n}}$时，应先通过梯度下降公式计算出代价函数，将所有的${\theta_{i}}$全部更新后，再重新更新代价函数。不要每计算出一个${\theta_{i}}$就更新一次代价函数。

(三)学习率的大小

为了使代价函数快速变到最小，我们需要合理选取$\alpha$值。

当$a$选取太小时，学习速率太小，变化速度会很慢。
当$a$选取太大时，梯度可能越过最低点，甚至可能无法收敛。
而合适的学习率应该使步长从较大慢慢减到较小，最后使代价函数逼近最低点。

而根据学习率不同，代价函数如何变化？一个好的学习率对应的代价函数应符合第一幅图中的变化情况：先快速更新到临近区域，接着逐渐逼近最佳结果。第二个为学习率过小情况，代价函数变化缓慢。第三个是学习率过大的情况，代价函数最终无法收敛。

那么学习率到底多少合适？这个并没有一个统一的答案，不同的训练集，不同的数据，会使得最佳学习率不同。因此在实际问题时，应该根据代价函数的变化情况对学习率进行不断调整，最后找到合适的学习率。
通常可以考虑尝试些学习率：$\alpha=0.01，0.03，0.1，0.3，1，3，10$

四、梯度下降与线性回归

讲完了梯度下降，现在将梯度下降与单变量线性回归相结合。梯度下降算法和线性回归算法比较如图：

对我们之前的线性回归问题运用梯度下降法，关键在于求出代价函数的导数，即：
$\frac{\partial }{\partial \theta _j} J \left( \theta_0, \theta_1 \right) = \frac{\partial }{\partial \theta j}\frac{1}{2m}\sum\limits{i=1}^m \left( h_{\theta}(x^{(i)})-y^{(i)} \right)^{2}$
$j=0 时：\frac{\partial }{\partial \theta _0} J \left( \theta_0, \theta_1 \right) = \frac{1}{m}\sum\limits_{i=1}^m \left( h_{\theta}(x^{(i)})-y^{(i)} \right)$
$j=1 时：\frac{\partial }{\partial \theta _1} J \left( \theta_0, \theta_1 \right) = \frac{1}{m}\sum\limits_{i=1}^{m}{\left( \left( {h_{\theta }}(x^{(i)})-{y^{(i)}} \right)\cdot x^{(i)} \right)}$
则算法改写成：
$\text{Repeat: } \lbrace$
$\theta_0 := \theta_0 - \alpha \frac{1}{m} \sum\limits_{i=1}^{m}(h_\theta(x_{i}) - y_{i})$
$\theta_1 := \theta_1 - \alpha \frac{1}{m}\sum\limits_{i=1}^{m}\left((h_\theta(x_{i}) - y_{i}) x_{i}\right)$
$\rbrace$
