输入: (2 -> 4 -> 3) + (5 -> 6 -> 4)

解释: 342 + 465 = 807.

输出单链表倒数第 K 个节点

问题描述

题目：输入一个单链表，输出此链表中的倒数第 K 个节点。（去除头结点，节点计数从 1 开始）。

输入: 1->2->3->4->5   K=2
输出: 1->2->3->5

LeetCode相关题目地址：19-Remove Nth Node From End of List

常见思路

思路一：两遍遍历法。

  1. 遍历单链表，遍历同时得出链表长度 N 。
  2. 再次从头遍历，访问至第 N - K 个节点为所求节点。
  • 时间复杂度O(2n)

思路二：递归法

  1. 定义num = k
  2. 使用递归方式遍历至链表末尾。
  3. 由末尾开始返回，每返回一次 num 减 1
  4. 当 num 为 0 时，即可找到目标节点
  • 时间复杂度O(2n)

思路三：双指针法

  1. 定义快慢两个指针，快指针先遍历
  2. 当快指针遍历了k个节点后，慢指针开始遍历
  3. 当快指针遍历至链表尾部时，慢指针与链表尾部相差k长度，即当前为目标节点
  • 时间复杂度O(n)

单链表与环Q1-环存在问题

问题描述

题目：输入一个单链表，输出此链表中是否存在环。

输入: head = [3,2,0,-4], pos = 1(connect to 1st)
输出: true
输入: head = [3,2,0,-4], pos = -1(no connection)
输出: false

LeetCode相关题目地址：141-Linked List Cycle

常见思路

思路一：哈希缓存比较法

  1. 遍历链表，记录已访问的节点。
  2. 将当前节点与之前以及访问过的节点比较，若有相同节点则有环。否则，不存在环。
  • 空间复杂度O(n)，时间复杂度O(n)

思路二：双指针法

  1. 定义两个指针分别为 slow，fast，并且将指针均指向链表头节点。
  2. 规定，slow 指针每次前进 1 个节点，fast 指针每次前进两个节点。
  3. 当 slow 与 fast 相等，且二者均不为空，则链表存在环。
  • 空间复杂度O(1)，时间复杂度O(n)

代码实现

单链表与环Q2-环的起始点

问题描述

题目：输入一个单链表，判断是否存在环。若存在则输出环的起始节点，反之为空

输入: head = [3,2,0,-4], pos = 1(connect to 1st)
输出: 1
输入: head = [3,2,0,-4], pos = -1(no connection)
输出: NULL

LeetCode相关题目地址：142-Linked List Cycle II

解题思想

基于上题中判断是否存在环的方法，本题继续采用双指针法的方法。

假定链表中存在环。易证slow与fast第一次相遇时，slow最多刚遍历一遍链表（链表尾节点与头节点相连）。设此时，slow移动距离为s，fast移动距离2s，fast在环中循环圈数n。设链表长度为l，环入口点距离头节点a，slow与fast首次相遇距离入口点b，环长度r。则有：

s = a + b;
2s = a + b + n*r;
那么可以得到s = n*r;
a + b = s = n*r = (n - 1)*r + l - a;
那么可以得到a = (n - 1)*r + (l - a - b);

注意到 (l - a - b) 即为第一次相遇点继续遍历到达环入口点的距离。因此，我们生成指针p1与p2. p1指针从相遇点开始遍历，p2从链表头开始遍历。当两个指针相等时，当前点即为入口点。两个指针分别走了(n-1)*r+(l-a-b)与a距离。

代码实现

有序链表合并Q2-合并多个单链表

问题描述

题目：将多个有序链表合并为一个新的有序链表并返回。新链表是通过拼接给定的两个链表的所有节点组成的。

输入：1->4->5,
            1->3->4,
            2->6,
输出：1->1->2->3->4->4->5->6

LeetCode相关题目地址：23-Merge k Sorted Lists

常见思路

思路一：一个个元素比较

  1. 通过k-1次比较，获得k个链表头节点的最小值
  2. 将最小值添加到返回链表中，并更新k个链表的头节点
  • 时间复杂度O(kN)，空间复杂度O(1)

思路二：最小堆/优先队列

  1. 生成一个k个元素的最小堆。每读入一个元素，更新最小堆并输出最小堆的堆顶元素
  • 时间复杂度O(Nkogk),空间复杂度O(k)

思路三：一个个链表合并

  1. 第一个与第二个链表合并，结果再与第三个链表合并…
  • 时间复杂度O(kN)，空间复杂度O(1)

思路四：分治合并

  1. k个链表两两进行合并。
  2. 对合并后的链表继续两两合并，直到仅剩一个链表。
  • 时间复杂度O(Nlogk)，空间复杂度O(1)
